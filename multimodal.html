1. Dataset Exploration & Preprocessing

Dataset Overview: MMIMDb

Source: Hugging Face - MMIMDb Dataset

The project utilizes the MMIMDb (Multi-Modal IMDb) dataset, a benchmark for multimodal classification tasks. The objective is to predict movie genres based on both visual information (movie posters) and textual information (plot summaries).

15,552 Total Samples
26 Unique Classes
Multi-label Problem Type

Data Characteristics:

  • Textual Data: The plot attribute provides a concise summary of the movie's content.
  • Visual Data: The images attribute contains the official movie posters.
  • Labels: Common genres include Drama (8,424), Comedy (5,108), Romance (3,226), etc.

Full Genre List (26 Classes):

Action, Adventure, Animation, Biography, Comedy, Crime, Documentary, Drama, Family, Fantasy, Film-Noir, History, Horror, Music, Musical, Mystery, News, Reality-TV, Romance, Sci-Fi, Short, Sport, Talk-Show, Thriller, War, Western.

Interactive Training Pipeline

Input (Image + Text)

In this initial stage, the system retrieves and structures the raw information required for multimodal learning.

  • Textual Input: The system extracts movie plot summaries from the dataset. This serves as the primary linguistic source for understanding the narrative context and semantic themes of each film.
  • Visual Input: Poster images are accessed via local file paths and opened using the PIL library. This provides the raw pixel data necessary for the vision encoder to capture aesthetic and symbolic genre cues.
  • Multi-label Binarization: Since movies often belong to multiple categories, genre labels are transformed using a MultiLabelBinarizer. This process converts text tags into binary vectors, allowing the model to treat the task as a series of simultaneous classification decisions rather than a single-choice problem.

Pre-process (Encoding & Transformation)

Raw inputs are transformed into numerical tensors through tokenizer and image processor modules, ensuring compatibility with multimodal backbones and stable batch training.

Image Resizing & Standardization

Convert images of different resolutions into a uniform format for efficient batch processing.

Objective: Ensure all samples produce consistent image tensors with shape (N,3,224,224).

  • Step 1 - RGB Conversion: Convert every poster to RGB before tensor conversion.
  • Step 2 - Resize: Resize to 224x224 (ViT-compatible resolution).
  • Step 3 - Processor Encoding: AutoImageProcessor converts image into pixel_values tensors.
  • Step 4 - Fallback Safety: If image files are missing or corrupted, use a blank image to prevent crash.
Text Encoding & Label Preparation
  • Text Tokenization: Plot summaries are encoded with DistilBERT tokenizer using max_length=128, truncation, and max-length padding.
  • Multi-label Binarization: Genre strings are converted to 26-dimensional binary vectors using MultiLabelBinarizer.
Data Split Strategy (Train/Val/Test)

We apply a deterministic 80/10/10 split to separate optimization, tuning, and final evaluation.

  • Train set (80%): used to update model parameters.
  • Validation set (10%): used for model selection and hyperparameter checks.
  • Test set (10%): reserved for final unbiased reporting.
  • Leakage control: all splits are created before DataLoader construction to avoid overlap.

Backbone (CLIP / Encoder)

The backbone is responsible for extracting high-dimensional semantic representations from each modality (image and text) before they are passed to the classification head. The backbone architecture differs across the three experimental settings.

In every setting (Zero-Shot, Few-Shot, and Full Fine-Tune), we run a controlled comparison across three modality variants: Text-Only, Image-Only, and Multimodal (Text + Image).

Zero-Shot — CLIPModel (Fully Frozen)

A single unified multimodal model is used as the backbone, handling both image and text feature extraction simultaneously.

  • Model: CLIPModel — checkpoint openai/clip-vit-base-patch32.
  • Vision encoder: ViT-Base with patch size 32×32 — converts the input poster into a 512-dim image embedding.
  • Text encoder: OpenAI's dedicated Transformer — encodes genre prompt strings into a 512-dim text embedding.
  • State: Fully frozen — no parameters are updated. The model is only called via model(**inputs) to retrieve embeddings.
  • Feature use: Cosine similarity between image and text embeddings determines the zero-shot genre prediction.
Few-Shot — CLIPModel (Frozen Feature Extractor)

The same CLIP model is reused but restructured as a fixed feature extractor feeding a trainable linear classifier.

  • Model: CLIPModel — checkpoint openai/clip-vit-base-patch32.
  • Internal branches: self.clip.vision_model (image) and self.clip.text_model (text) are accessed separately.
  • State: Fully frozen — param.requires_grad = False is applied to all CLIP parameters. Only the downstream nn.Sequential classifier is trained.
  • Training data: The linear probe is trained on 5% of the dataset to simulate a low-resource few-shot scenario.
Full Fine-Tune — DistilBERT + ViT (Unfrozen)

Two separate backbone models from different model families are used — one for each modality — and both are fully updated during training.

  • Text backbone: DistilBERT (distilbert-base-uncased) loaded via AutoModel.
  • Image backbone: ViT-Base (google/vit-base-patch16-224-in21k) loaded via ViTModel.
  • State: Fully unfrozen — model.parameters() is passed directly to the optimizer, so all weights of both DistilBERT and ViT are updated to specialize on MM-IMDb.
  • Feature fusion: The [CLS] token of DistilBERT and the pooled output of ViT are concatenated and forwarded to the multi-label classification head.

Head (Zero-shot/Few-shot/Full Fine-tune)

The head maps extracted features to 26 multi-label logits and uses a strategy-specific training setup for each mode.

For each strategy, the same evaluation protocol is applied to three variants: Text-Only, Image-Only, and Multimodal (Text + Image), so the head behavior can be compared fairly across modalities.

Zero-Shot Head (Similarity-based Scoring)

The "Head" in the zero-shot configuration is a non-parametric feature comparator. It leverages CLIP’s pre-trained joint latent space to perform classification by measuring the semantic alignment between inputs and label descriptions without any weight updates.

  • Mechanism: The head calculates the Cosine Similarity between the sample embeddings (either the movie poster or the plot text) and the embeddings of 26 natural language prompts structured as: "A movie poster of the genre [Genre Name]".
  • Fusion Strategy (Multimodal): It employs Late Fusion (Probability-level). The system independently computes Softmax probabilities for the Image-to-Genre and Text-to-Genre branches, then applies Mean Fusion (averaging the probabilities) to reach a final consensus.
  • Decision Rule: Since MM-IMDb is a multi-label task, the head uses a fixed threshold of 0.1 rather than a Top-k policy. Any genre with a similarity probability exceeding this threshold is assigned as a label for the movie.
Few-Shot Head (Linear Probe)

Backbone features are frozen and only the lightweight classifier is trained with BCEWithLogitsLoss.

  • Head structure: Linear/MLP classifier over frozen CLIP features.
  • Optimization: AdamW with low learning rate for stable few-shot adaptation.
  • Decision rule: torch.sigmoid(logits) > 0.2.
Full Fine-Tune Head (Trainable End-to-End Classifier)

In this strategy, both backbone transformers (DistilBERT and ViT) and the classification heads are trained simultaneously using a low learning rate (2e-5) to adapt to the MM-IMDb domain.

  • Feature Extraction: The head uses the [CLS] representation from both models:
    • Text = last_hidden_state[:, 0, :] (768-dim)
    • Image = last_hidden_state[:, 0, :] (768-dim)
  • Modality-Specific Heads: For unimodal runs (Text-Only or Image-Only), the classifier is a single nn.Linear(768, C) layer.
  • Multimodal Fusion Head: Concatenate features (768 + 768 = 1536).
  • Classifier Architecture: Linear(1536 → 512) → ReLU → Dropout(0.2) → Linear(512 → C)
  • Optimization & Decision: Use BCEWithLogitsLoss; predictions are generated with torch.sigmoid(logits) > 0.5.
  • Metrics: Micro-F1, Macro-F1, and Subset Accuracy.

4. Error Analysis

To gain a deeper understanding of the model's limitations, we performed both quantitative and qualitative error analyses on the validation set. We specifically focused on "Critical Failures," defined as samples where the Jaccard Similarity (the overlap between predicted and true genres) was below 0.2.

4.1 Quantitative Analysis: Critical Error Rate

Based on our evaluation of the validation set (approximately 1,520 samples):

  • Total Critical Errors (Jaccard < 0.2): 152 samples.
  • Critical Error Rate: ~10.0%.
Observation: While the model achieves a high overall Micro-F1 (0.66), 1 in 10 movies results in a near-total misclassification. This indicates that despite general accuracy, the model remains susceptible to strong "distractors" in either the textual or visual modality.

4.2 Qualitative Case Study: Keyword Bias & Visual Ambiguity

We analyzed a representative case of critical failure to understand the interplay between textual and visual features.

Analysis of Failure Modes:

  1. Keyword Bias (Textual Over-reliance): The plot includes phrases like "South Korean underworld" and "defeating an army of gangsters", which are strongly associated with Crime and Thriller genres in the training data. As a result, the model overemphasizes these keywords, failing to capture the comedic and romantic elements that define the film's true genre.
  2. Visual Ambiguity (Visual Misinterpretation): Action-comedy posters often feature high-contrast lighting and weaponry to attract attention. The Vision Transformer (ViT) interprets these cues similarly to those in serious Crime/Drama films, as it lacks the ability to recognize subtle indicators of comedy, such as exaggerated expressions or vibrant color tones.

4.3 The Overconfidence Problem

A striking finding was the model's high confidence in its errors. In the case above, the model predicted Drama with a confidence score of 92.62%, despite it being incorrect.

Model Over-calibration

Modern architectures trained with BCE Loss tend to be "over-confident." The loss function penalizes the model if it doesn't push probabilities toward 1 or 0, leading to aggressive maximization of a class if a single strong feature (like the word "gangster") is detected.

"Safe Bet" Strategy

Drama appears in nearly 50% of the MMIMDb dataset. The model learns that predicting "Drama" for any narrative with emotional weight is a statistically safe bet, leading to a systemic bias toward this label with high certainty.

4.4 Systemic Patterns from Quantitative Data

The following visualizations illustrate the distribution of errors across genres and the density of critical failures:

📊 Fig 4.4a: Analysis of Critical Prediction Errors by Genre

Comedy, Thriller, and Romance show the highest False Negatives. These genres rely on subtle emotional and narrative cues that the late-fusion approach struggles to capture.

📈 Fig 4.4b: Distribution of Prediction Overlap in Failed Samples

Many critical errors fall within the 0.0 - 0.1 Jaccard score range, representing complete prediction failures where one modality's misleading signals overpower the other's correct cues.

  • Tonal Blindness: Comedy, Thriller, and Romance show the highest False Negatives. Unlike genres such as "Action" or "Horror" with clear visual and textual signals, these rely on subtle emotional and narrative cues. The current late-fusion approach (DistilBERT + ViT concatenation) struggles to capture these nuances, often missing them in predictions.
  • The "Safe Bet" Bias: There is a significant spike in False Positives for the Drama category. Due to class imbalance—where Drama dominates the dataset—the model adopts a “safe bet” strategy, frequently defaulting to this label when inputs are ambiguous.
  • Modality Disconnect: Many critical errors fall within the 0.0 - 0.1 Jaccard score range, representing complete prediction failures. In these cases, misleading signals from one modality (e.g., strong keywords or visuals) overpower correct cues from the other, exposing the weakness of simple feature concatenation in handling cross-modal interactions.

7. Conclusion

In this assignment, we developed and evaluated multiple approaches for movie genre classification using the MMIMDb dataset, ranging from zero-shot multimodal heuristics to full end-to-end fine-tuning of Vision Transformer (ViT) and DistilBERT architectures.

Key Findings & Insights

Multimodal Superiority

The fully fine-tuned fusion model (Micro-F1: 0.6624) significantly outperformed the best Zero-shot baseline (Micro-F1: 0.3219).

This confirms that movie genres are often metaphorical and context-dependent, requiring the deep parameter updates of specialized Transformers to capture nuanced relationships that general-purpose alignments miss.

Efficiency of Few-Shot Learning

By training on only 5% of the data (622 samples), the Few-Shot Linear Probe achieved 89% of the performance of the full fine-tuned version (Micro-F1: 0.5908).

This demonstrates a highly cost-effective strategy for labeling niche cinematic databases where large-scale human annotation is not feasible, yet high precision is required.
