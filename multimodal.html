Since movies often belong to multiple categories simultaneously, this project is framed as a multi-label classification task. To handle this, we utilized the MultiLabelBinarizer from the scikit-learn library.

Final Output: Each sample is assigned a ground-truth vector of size N=26 (the total number of unique genres in the MMIMDb dataset), where a 1 indicates the presence of a genre and 0 indicates its absence.

Stage 2: Custom Multimodal PyTorch Dataset

We defined the MMIMDbLocalDataset class to perform on-the-fly transformations. It uses DistilBERT Tokenizer for text and ViT Image Processor for visual data. 


class MMIMDbLocalDataset(Dataset):
    def __init__(self, df, tokenizer, image_processor, max_len=128):
        self.df = df.reset_index(drop=True)
        self.tokenizer = tokenizer
        self.image_processor = image_processor
        self.max_len = max_len

Processing Workflow:

  • Text Tokenization: Plot summaries are processed using the AutoTokenizer from the distilbert-base-uncased model. We apply a fixed max_length of 128 tokens, utilizing truncation=True and padding='max_length' to ensure uniform batch shapes.
  • Image Feature Extraction: Posters are prepared via the AutoImageProcessor for ViT. This involves resizing the images to 224x224 pixels and normalizing the pixel values based on the ImageNet mean and standard deviation.
  • Error Handling: A try-except block is implemented within the __getitem__ method. In the event of a corrupted image file, the system automatically generates a blank "fallback" tensor to prevent the training loop from crashing. 

# Dataset __getitem__ Implementation:
def __getitem__(self, idx):
    row = self.df.iloc[idx]
        
    # --- Text Processing ---
    text_enc = self.tokenizer(
        str(row['plot']), 
        truncation=True, 
        max_length=self.max_len, 
        padding='max_length', 
        return_tensors='pt'
    )
    
    # --- Image Processing (Loading from local file path) ---
    try:
        image = Image.open(row['image_path']).convert('RGB')
        # ViT requires 224x224 input size
        image = image.resize((224, 224))
        image_enc = self.image_processor(images=image, return_tensors="pt")
    except Exception as e:
        # Fallback if the image file is corrupted or missing
        image = Image.new('RGB', (224, 224))
        image_enc = self.image_processor(images=image, return_tensors="pt")
    
    # --- Label Handling ---
    labels = torch.tensor(row['labels_binary'], dtype=torch.float)
    
    return {
        'input_ids': text_enc['input_ids'].squeeze(0),
        'attention_mask': text_enc['attention_mask'].squeeze(0),
        'pixel_values': image_enc['pixel_values'].squeeze(0),
        'labels': labels
    }

6. Advanced Multimodal & Prompt-based Methodology

To push the boundaries of movie genre classification, we implemented three distinct advanced strategies: a Full Fine-Tuning Multimodal Fusion model, Zero-Shot Classification with CLIP, and Few-Shot Linear Probing.

6.1 Multimodal Full Fine-Tuning (DistilBERT + ViT)

Our primary architecture processes textual plots and visual posters simultaneously to achieve maximum predictive accuracy by allowing cross-modal feature interaction.

Architectural Design & Strategy

  • Multimodal Architecture: To achieve high accuracy, we designed a deep multimodal model that processes both textual plots and visual posters simultaneously.
  • Dual Encoders: We integrated DistilBERT (text branch) and ViT-Base (image branch) into a unified pipeline. Each produces a 768-dimensional representation, which are concatenated into a 1536-dimensional joint feature vector.
  • Fusion Head: The joint vector is passed through a classification head: Linear (512) → ReLU → Dropout (0.2) → Output (26 classes).
  • Training Strategy: We apply full fine-tuning, allowing gradients to update both backbones. The model is trained using AdamW (learning rate: 2e-5) for 5 epochs.
Implementation: Multimodal Architecture 

# 1. Import the Dataset class (Ensure split_data_and_dataloader.py is in the same directory)
from split_data_and_dataloader import MMIMDbLocalDataset

# 2. Model Architecture Definition
class MultimodalModel(nn.Module):
  def __init__(self, num_classes):
    super(MultimodalModel, self).__init__()
    # Text Backbone: DistilBERT
    self.text_model = AutoModel.from_pretrained("distilbert-base-uncased")
    # Image Backbone: Vision Transformer (ViT)
    self.image_model = ViTModel.from_pretrained("google/vit-base-patch16-224-in21k")
    
    # Late Fusion Classifier
    # Concatenates 768 (Text) + 768 (Image) features = 1536 input features
    self.classifier = nn.Sequential(
      nn.Linear(768 + 768, 512),
      nn.ReLU(),
      nn.Dropout(0.2),
      nn.Linear(512, num_classes)
    )

  def forward(self, input_ids, attention_mask, pixel_values):
    # Extract text features from the [CLS] token (index 0)
    text_out = self.text_model(input_ids=input_ids, attention_mask=attention_mask)
    text_feat = text_out.last_hidden_state[:, 0, :] 
    
    # Extract image features from the [CLS] token
    img_out = self.image_model(pixel_values=pixel_values)
    img_feat = img_out.last_hidden_state[:, 0, :] 
    
    # Concatenate modality features (Late Fusion)
    combined = torch.cat((text_feat, img_feat), dim=1)
    
    # Final classification
    return self.classifier(combined)

# 3. Evaluation Function
def evaluate(model, dataloader, criterion, device):
  model.eval()
  all_labels, all_preds = [], []
  val_loss = 0
  with torch.no_grad():
    for batch in dataloader:
      input_ids, attention_mask = batch['input_ids'].to(device), batch['attention_mask'].to(device)
      pixel_values, labels = batch['pixel_values'].to(device), batch['labels'].to(device)
      
      logits = model(input_ids, attention_mask, pixel_values)
      loss = criterion(logits, labels)
      val_loss += loss.item()
      
      # Multi-label prediction using sigmoid and a 0.5 threshold
      preds = (torch.sigmoid(logits) > 0.5).cpu().numpy()
      all_labels.append(labels.cpu().numpy())
      all_preds.append(preds)
          
  all_labels = np.vstack(all_labels)
  all_preds = np.vstack(all_preds)
  
  return (
    val_loss / len(dataloader), 
    f1_score(all_labels, all_preds, average='micro'), 
    f1_score(all_labels, all_preds, average='macro'), 
    accuracy_score(all_labels, all_preds)
  )

6.2 Zero-Shot Classification with CLIP

We utilized CLIP (Contrastive Language-Image Pre-training) to evaluate the model's ability to generalize to movie genres without any domain-specific training.

Mechanism & Inference

  • Zero-Shot Evaluation: To assess generalization without domain-specific training, we utilized CLIP (Contrastive Language-Image Pre-training) from OpenAI.
  • Mechanism: CLIP is pre-trained on 400 million image-text pairs, enabling it to learn visual concepts through natural language supervision.
  • Prompting Strategy: Classification is performed by computing cosine similarity between the poster and 26 text prompts, such as "A movie poster of the genre [Genre]".
  • Inference: The model produces a probability distribution over all prompts, and labels are assigned using a similarity threshold of 0.1.
Implementation: CLIP Zero-Shot 

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
print(f">>> Running Zero-shot inference on: {device}")

# Load metadata to retrieve the standard list of genres
df = pd.read_csv("mmimdb_metadata.csv")
df['genres'] = df['genres'].apply(lambda x: x.split("|") if isinstance(x, str) else [])

# Initialize MultiLabelBinarizer to handle 26 movie genres
mlb = MultiLabelBinarizer()
mlb.fit(df['genres'])
class_names = mlb.classes_ 

# Isolate the Test set (10% subset, maintaining consistency with your previous splits)
_, test_indices = torch.utils.data.random_split(
    df, [int(0.9 * len(df)), len(df) - int(0.9 * len(df))], 
    generator=torch.Generator().manual_seed(42)
)
test_df = df.iloc[test_indices.indices]

# 2. Initialize CLIP Model
# We use the ViT-B/32 variant, which is balanced for speed and accuracy
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32").to(device)
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

# Prepare natural language prompts for CLIP
# Using a descriptive phrase like "A movie poster of the genre..." helps CLIP align 
# visual features with textual concepts more effectively than using single words.
prompts = [f"A movie poster of the genre {g}" for g in class_names]

all_labels = []
all_preds = []

# 3. Perform Inference
model.eval()
print(">>> Starting Zero-shot prediction process...")

with torch.no_grad():
    for _, row in tqdm(test_df.iterrows(), total=len(test_df)):
        # Load and convert image to RGB
        try:
            image = Image.open(row['image_path']).convert('RGB')
        except Exception:
            # Fallback for missing/corrupted images
            image = Image.new('RGB', (224, 224), color=0)
        
        # Preprocess both the image and the text prompts
        inputs = processor(
            text=prompts, 
            images=image, 
            return_tensors="pt", 
            padding=True
        ).to(device)

        # CLIP calculates the cosine similarity between the image embedding and prompt embeddings
        outputs = model(**inputs)
        logits_per_image = outputs.logits_per_image  # Similarity scores
        probs = logits_per_image.softmax(dim=-1).cpu().numpy()[0]

        # Multi-label Strategy:
        # Since the sum of probabilities across 26 classes equals 1.0, 
        # a threshold of 0.1 is used to select the most likely genres.
        pred_binary = (probs > 0.1).astype(int) 
        
        # Transform ground truth labels into binary format
        true_binary = mlb.transform([row['genres']])[0]
        
        all_labels.append(true_binary)
        all_preds.append(pred_binary)

# 4. Results Evaluation
all_labels = np.array(all_labels)
all_preds = np.array(all_preds)

micro_f1 = f1_score(all_labels, all_preds, average='micro', zero_division=0)
macro_f1 = f1_score(all_labels, all_preds, average='macro', zero_division=0)
acc = accuracy_score(all_labels, all_preds)

print(f"\n--- ZERO-SHOT (CLIP) BASELINE RESULTS ---")
print(f"Micro F1: {micro_f1:.4f}")
print(f"Macro F1: {macro_f1:.4f}")
print(f"Subset Accuracy (Exact Match): {acc:.4f}")

print("\nPer-class Classification Report:\n")
print(classification_report(all_labels, all_preds, target_names=class_names, zero_division=0))

6.3 Few-Shot Linear Probing with CLIP

We explored a Few-Shot approach to observe how effectively the model learns from extremely limited labeled data while minimizing computational costs.

Data Efficiency & Optimization

  • Few-Shot Strategy: To reduce computational cost, we adopted a few-shot approach to evaluate performance with limited training data.
  • Data Efficiency: Only 5% of the training set (approximately 500–600 samples) was used to train a lightweight classifier.
  • Linear Probe: The CLIP visual backbone (ViT-B/32) was frozen, and only a single Linear layer (the probe) was trained on top of its extracted features.
  • Optimization: This approach is computationally efficient, leveraging strong pre-trained representations without full model updates. Training was conducted for 10 epochs using a learning rate of 1e-3.
Implementation: CLIP Linear Probe 

class CLIPImageDataset(Dataset):
    def __init__(self, df, processor):
        self.df = df.reset_index(drop=True)
        self.processor = processor

    def __len__(self):
        return len(self.df)

    def __getitem__(self, idx):
        row = self.df.iloc[idx]
        try:
            image = Image.open(row['image_path']).convert('RGB')
            inputs = self.processor(images=image, return_tensors="pt")
        except Exception:
            # Fallback if the image is corrupted or missing
            image = Image.new('RGB', (224, 224))
            inputs = self.processor(images=image, return_tensors="pt")
            
        labels = torch.tensor(row['labels_binary'], dtype=torch.float)
        return inputs['pixel_values'].squeeze(0), labels

# 2. Linear Probe Model Definition
class CLIPLinearProbe(nn.Module):
    def __init__(self, clip_model, num_classes):
        super().__init__()
        # Freeze CLIP vision backbone
        self.clip_visual = clip_model.vision_model 
        for param in self.clip_visual.parameters():
            param.requires_grad = False
        
        # Dynamically retrieve hidden_size from CLIP config
        self.classifier = nn.Linear(self.clip_visual.config.hidden_size, num_classes) 

    def forward(self, pixel_values):
        outputs = self.clip_visual(pixel_values)
        features = outputs.pooler_output 
        return self.classifier(features)

# 3. Multi-label Confusion Matrix (Heatmap) Plotting
def plot_multilabel_confusion_matrix(all_labels, all_preds, class_names):
    plt.figure(figsize=(15, 12))
    
    # Calculate correlation between ground truth and predicted labels
    confusion_grid = np.zeros((len(class_names), len(class_names)))
    for i in range(len(all_labels)):
        true_indices = np.where(all_labels[i] == 1)[0]
        pred_indices = np.where(all_preds[i] == 1)[0]
        for t in true_indices:
            for p in pred_indices:
                confusion_grid[t, p] += 1

    # Row-wise normalization
    row_sums = confusion_grid.sum(axis=1, keepdims=True)
    row_sums[row_sums == 0] = 1
    confusion_grid_norm = confusion_grid / row_sums

    sns.heatmap(confusion_grid_norm, annot=False, cmap='Blues', 
                xticklabels=class_names, yticklabels=class_names)
    plt.xlabel('Predicted Genres', fontsize=12)
    plt.ylabel('True Genres', fontsize=12)
    plt.title('Normalized Multi-label Confusion Heatmap (Few-shot CLIP)', fontsize=15)
    plt.tight_layout()
    plt.savefig('confusion_heatmap.png')
    plt.show()
    print(">>> Confusion matrix saved as confusion_heatmap.png")

7.2 Discussion of Findings

The Power of Multimodality: The Multimodal Full Fine-tuning model achieved the best performance, with a Micro-F1 of 0.6605 and a Macro-F1 of 0.4868. This demonstrates that combining textual plot summaries with visual poster features provides a more comprehensive understanding of movie genres than relying on a single modality.

Zero-Shot Limitations: The Zero-shot CLIP model showed significantly lower performance (0.2233). This indicates that while CLIP captures general visual concepts well, movie genre classification is highly nuanced, as posters often rely on abstract or metaphorical imagery that requires domain-specific context.

Few-Shot Efficiency: The Few-Shot Linear Probe, trained on only 5% of the dataset, achieved an F1-score of 0.5266—nearly matching the ResNet50 model trained on the full dataset. This highlights the effectiveness of leveraging strong pre-trained features in data-scarce scenarios.

Keyword Bias (Textual Over-reliance): The plot includes phrases like "South Korean underworld" and "defeating an army of gangsters", which are strongly associated with Crime and Thriller genres in the training data. As a result, the model overemphasizes these keywords, failing to capture the comedic and romantic elements that define the film’s true genre.
  2. Visual Ambiguity (Visual Misinterpretation): Action-comedy posters often feature high-contrast lighting and weaponry to attract attention. The Vision Transformer (ViT) interprets these cues similarly to those in serious Crime/Drama films, as it lacks the ability to recognize subtle indicators of comedy, such as exaggerated expressions or vibrant color tones.

8.3 The Overconfidence Problem

A striking finding was the model’s high confidence in its errors. In the case above, the model predicted Drama with a confidence score of 92.62%, despite it being incorrect.

Model Over-calibration

Modern architectures trained with BCE Loss tend to be "over-confident." The loss function penalizes the model if it doesn't push probabilities toward 1 or 0, leading to aggressive maximization of a class if a single strong feature (like the word "gangster") is detected.

"Safe Bet" Strategy

Drama appears in nearly 50% of the MMIMDb dataset. The model learns that predicting "Drama" for any narrative with emotional weight is a statistically safe bet, leading to a systemic bias toward this label with high certainty.

8.4 Systemic Patterns from Quantitative Data

Fig 8.4b: Jaccard Score Density (Critical Failures)

  • Tonal Blindness: Comedy, Thriller, and Romance show the highest False Negatives. Unlike genres such as "Action" or "Horror" with clear visual and textual signals, these rely on subtle emotional and narrative cues. The current late-fusion approach (DistilBERT + ViT concatenation) struggles to capture these nuances, often missing them in predictions.
  • The "Safe Bet" Bias: There is a significant spike in False Positives for the Drama category. Due to class imbalance—where Drama dominates the dataset—the model adopts a “safe bet” strategy, frequently defaulting to this label when inputs are ambiguous.
  • Modality Disconnect: Many critical errors fall within the 0.0–0.1 Jaccard score range, representing complete prediction failures. In these cases, misleading signals from one modality (e.g., strong keywords or visuals) overpower correct cues from the other, exposing the weakness of simple feature concatenation in handling cross-modal interactions.

12. Conclusion

In this assignment, we developed and evaluated multiple approaches for movie genre classification using the MMIMDb dataset, ranging from zero-shot heuristics to full multimodal fine-tuning.

Key Findings & Insights

Multimodal Superiority

The fine-tuned fusion model (Micro-F1: 0.6605) significantly outperformed the Zero-shot CLIP baseline (Micro-F1: 0.2233).

This confirms that movie genres are often metaphorical and context-dependent, requiring domain-specific fine-tuning rather than relying solely on general-purpose vision-language alignments.

Efficiency of Few-Shot Learning

By training on only 5% of the data, the Few-Shot Linear Probe achieved over 80% of the performance of the full fine-tuned version (Micro-F1: 0.5266).

This demonstrates a cost-effective and computationally efficient strategy for labeling niche or smaller cinematic databases where large-scale labeling is not feasible.
