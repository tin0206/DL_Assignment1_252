1. Dataset Exploration & Preprocessing

Dataset Overview: MMIMDb

Source: Hugging Face - MMIMDb Dataset

The project utilizes the MMIMDb (Multi-Modal IMDb) dataset, a benchmark for multimodal classification tasks. The objective is to predict movie genres based on both visual information (movie posters) and textual information (plot summaries).

15,552 Total Samples
26 Unique Classes
Multi-label Problem Type

Data Characteristics:

  • Textual Data: The plot attribute provides a concise summary of the movie's content.
  • Visual Data: The images attribute contains the official movie posters.
  • Labels: Common genres include Drama (8,424), Comedy (5,108), Romance (3,226), etc.

3. Dataset & DataLoader Setup

To build an efficient training pipeline, we implemented a two-stage process: first, localizing the dataset for faster access, and second, creating a custom PyTorch Dataset to handle multimodal inputs (text and images) simultaneously.

Stage 1: Data Localization & Pre-processing

Raw data from Hugging Face is processed and stored locally. This stage involves resizing images to a base size of 448x448 and extracting movie plot and list of genres of each sample into a persistent CSV metadata file. This ensures the training loop doesn't depend on network latency.

Since movies often belong to multiple categories simultaneously, this project is framed as a multi-label classification task. To handle this, we utilized the MultiLabelBinarizer from the scikit-learn library.

Final Output: Each sample is assigned a ground-truth vector of size N=26 (the total number of unique genres in the MMIMDb dataset), where a 1 indicates the presence of a genre and 0 indicates its absence.

Stage 2: Custom Multimodal PyTorch Dataset

We defined the MMIMDbLocalDataset class to perform on-the-fly transformations. It uses DistilBERT Tokenizer for text and ViT Image Processor for visual data. 


class MMIMDbLocalDataset(Dataset):
    def __init__(self, df, tokenizer, image_processor, max_len=128):
        self.df = df.reset_index(drop=True)
        self.tokenizer = tokenizer
        self.image_processor = image_processor
        self.max_len = max_len

Processing Workflow:

  • Text Tokenization: Plot summaries are processed using the AutoTokenizer from the distilbert-base-uncased model. We apply a fixed max_length of 128 tokens, utilizing truncation=True and padding='max_length' to ensure uniform batch shapes.
  • Image Feature Extraction: Posters are prepared via the AutoImageProcessor for ViT. This involves resizing the images to 224x224 pixels and normalizing the pixel values based on the ImageNet mean and standard deviation.
  • Error Handling: A try-except block is implemented within the __getitem__ method. In the event of a corrupted image file, the system automatically generates a blank "fallback" tensor to prevent the training loop from crashing. 

# Dataset __getitem__ Implementation:
def __getitem__(self, idx):
    row = self.df.iloc[idx]
        
    # --- Text Processing ---
    text_enc = self.tokenizer(
        str(row['plot']), 
        truncation=True, 
        max_length=self.max_len, 
        padding='max_length', 
        return_tensors='pt'
    )
    
    # --- Image Processing (Loading from local file path) ---
    try:
        image = Image.open(row['image_path']).convert('RGB')
        # ViT requires 224x224 input size
        image = image.resize((224, 224))
        image_enc = self.image_processor(images=image, return_tensors="pt")
    except Exception as e:
        # Fallback if the image file is corrupted or missing
        image = Image.new('RGB', (224, 224))
        image_enc = self.image_processor(images=image, return_tensors="pt")
    
    # --- Label Handling ---
    labels = torch.tensor(row['labels_binary'], dtype=torch.float)
    
    return {
        'input_ids': text_enc['input_ids'].squeeze(0),
        'attention_mask': text_enc['attention_mask'].squeeze(0),
        'pixel_values': image_enc['pixel_values'].squeeze(0),
        'labels': labels
    }

4. Image-only Baseline Models

To establish a robust baseline for visual feature extraction, we implemented and compared two state-of-the-art architectures. This comparison helps determine whether local spatial features (CNN) hoặc global context (Transformers) are more effective for decoding movie genres from posters.

4.1 Architectural Selection: ResNet50 vs. ViT-Base

ResNet50 (CNN)

A 50-layer deep Residual Network that utilizes convolutional layers and skip connections. It is highly efficient at capturing local patterns like shapes, textures, and specific objects within a poster.

ViT-Base (Vision Transformer)

Treats an image as a sequence of patches (16x16 pixels). Using the Self-Attention mechanism, ViT captures global dependencies, which is crucial for understanding the overall artistic composition and mood.

4.2 Training Methodology

Both models were initialized with Pre-trained Weights from ImageNet-1K. Due to the multi-label nature of the task, we utilized specific configurations:

  • Optimization: AdamW optimizer with learning rates of 1e-4 (ResNet) and 5e-5 (ViT).
  • Loss Function: BCEWithLogitsLoss, treating each genre as an independent binary classification.
  • Thresholding: A custom threshold of 0.35 was applied to account for the skewed genre distribution, prioritizing the recall of less frequent genres.

5. Text-only Baseline Models

Movie plot summaries in the MMIMDb dataset vary significantly in length and narrative complexity. We evaluated two distinct approaches to process this textual information: a sequential Recurrent Neural Network (RNN) and an attention-based Transformer model.

5.1 Architectural Selection: Bi-LSTM vs. DistilBERT

Bidirectional LSTM (RNN)

Processes text in both forward and backward directions to capture sequential dependencies. We implemented a custom SimpleTokenizer with a vocabulary of 20,000 words and 100-dimensional embeddings.

DistilBERT (Transformer)

A lighter, faster version of BERT that utilizes Self-Attention to understand relationships between all words simultaneously. This allows for a deeper grasp of semantic context compared to sequential processing.

5.1.1 Text Baseline Implementation (Source Code)

Bi-LSTM Classifier (RNN)


        class LSTMClassifier(nn.Module):
            def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
                super(LSTMClassifier, self).__init__()
                self.embedding = nn.Embedding(vocab_size, embed_dim)
                # Bidirectional LSTM captures context from both directions
                self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True, bidirectional=True)
                # Final layer input is hidden_dim * 2 due to bidirectionality
                self.fc = nn.Linear(hidden_dim * 2, num_classes)
                self.dropout = nn.Dropout(0.3)

            def forward(self, x):
                x = self.embedding(x)
                _, (hidden, _) = self.lstm(x)
                # Concatenate final hidden states from forward and backward passes
                cat_hidden = torch.cat((hidden[-2,:,:], hidden[-1,:,:]), dim=1)
                return self.fc(self.dropout(cat_hidden))

DistilBERT Classifier (Transformer)


        class TextClassifier(nn.Module):
            def __init__(self, num_classes):
                super(TextClassifier, self).__init__()
                self.distilbert = AutoModel.from_pretrained("distilbert-base-uncased")
                self.classifier = nn.Sequential(
                    nn.Linear(768, 512),
                    nn.ReLU(),
                    nn.Dropout(0.2),
                    nn.Linear(512, num_classes)
                )

            def forward(self, input_ids, attention_mask):
                out = self.distilbert(input_ids=input_ids, attention_mask=attention_mask)
                # Extract the [CLS] token (index 0) for classification
                cls_feat = out.last_hidden_state[:, 0, :]
                return self.classifier(cls_feat)

5.2 Training Methodology

  • Bi-LSTM: Trained with Adam (LR: 1e-3), hidden dimension 256, and dropout 0.3.
  • DistilBERT: Trained with AdamW (LR: 5e-5) to preserve pre-trained knowledge.
  • Common: 10 epochs, BCEWithLogitsLoss, and max_length=128.
