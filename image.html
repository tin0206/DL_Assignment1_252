1. Dataset Exploration & Preprocessing

Dataset Overview: CIFAR10

Source: Hugging Face - CIFAR10 Dataset

The project utilizes the CIFAR10 (Canadian Institute For Advanced Research - 10) dataset, a standard benchmark for image classification tasks. The primary objective is to evaluate and compare the performance of Convolutional Neural Networks (CNN) and Vision Transformer (ViT) architectures.

60,000 (50,000 train + 10,000 test) Total Samples
10 Unique Classes
Visual Information (Image) Modalities

Data Characteristics:

  • Color Profile: Pixel intensity analysis shows a non-uniform distribution across channels, specifically a high frequency in the Blue channel (~70 intensity), likely due to sky or water backgrounds.
  • Visual Data: The img attribute consists of color images with a native resolution of 32x32 pixels, representing 10 distinct object categories.
  • Label Distribution: The dataset is perfectly balanced, with each of the 10 classes containing exactly 6,000 samples (5,000 for training and 1,000 for testing).

Full Class List (10 Classes):

Airplane, Automobile, Bird, Cat, Deer, Dog, Frog, Horse, Ship, Truck.

Class Distribution Analysis

What:

Examine the number of samples across all 10 categories to ensure a uniform distribution within the CIFAR-10 dataset for both training and testing sets.

Why:

Class imbalance can heavily bias the model's predictions toward majority classes (e.g., assuming everything is a "Frog" if frogs dominate the data). A perfectly balanced dataset ensures the model learns distinctive features for every class equally, preventing skewed accuracy metrics.

How:

Calculate the frequency of each class label, verify that each contains exactly 5,000 training and 1,000 testing samples, and visualize the results using a bar chart to confirm the lack of a "long-tail" distribution.

As we can see in Figure 2 and 3, the horizontal 'flat' line across all bars confirms that our dataset is perfectly balanced. This provides a solid foundation for fair evaluation across all organic and inorganic categories.

Pixel Analysis

What:

Analyze the raw pixel intensity distribution across the Red, Green, and Blue (RGB) channels to identify global color biases and brightness patterns within the CIFAR-10 images.

Why:

Color distributions reveal fundamental characteristics of the dataset environment, such as the dominance of outdoor backgrounds (sky/sea). Understanding this non-uniformity provides the scientific justification for applying per-channel Normalization to ensure the model focuses on structural features rather than lighting or color shifts.

How:

Aggregate the pixel values (0-255) for each channel from the entire dataset, compute their frequency, and visualize the results using a histogram. Calculate the specific mean and standard deviation for each channel to use as input parameters for the pre-processing normalization layer.

Our analysis of raw pixel intensities across RGB channels reveals a dominant Blue channel (~70 intensity), suggesting a high frequency of cool-toned backgrounds (sky, water). A sharp spike at 255 across all channels indicates saturated regions or bright highlights.

Co-occurrence Analysis

What:

Quantify the visual and structural relationships between different categories by calculating the Cosine Similarity between their raw 32 x 32 pixel vectors.

Why:

High similarity scores (near 1.0) indicate "Semantic Overlap," where different classes share nearly identical pixel patterns. Identifying these clusters—such as the Animal cluster (0.998 similarity) — allows us to predict and justify specific model failures (misclassifications) before the training even begins.

How:

Compute the mean pixel vector for each class, perform pairwise Cosine Similarity calculations to build a relationship matrix, and visualize the findings using a Heatmap. This statistical proximity provides the empirical evidence needed to justify using deep architectures like ResNet50 for complex feature separation.

The correlation matrix (or Similarity Heatmap) visualizes how certain classes share visual features. Significant overlap is observed in the Animal cluster (e.g., Cat often shares features with Dog) and the Vehicle cluster (Automobile vs. Truck).

3. System Overview

Interactive Training Pipeline

Input + Pre-process

This stage establishes a robust flow from raw data ingestion to model-ready tensors. By integrating Data Ingestion and Pre-processing, we ensure that each image is dynamically transformed according to the specific architectural requirements of the chosen model.

  • Dataset Integration: We utilize the Hugging Face datasets library to stream the CIFAR-10 dataset directly, enabling efficient handling of 60,000 (32 x 32) images.
  • Split Management: To ensure unbiased performance monitoring, the 50,000-image training set is partitioned into a 90% Training subset and a 10% Validation subset. The 10,000-image test set remains isolated for final benchmarking.
  • ResNet50 (CNN): Employs Random Cropping and Horizontal Flipping for spatial augmentation, followed by normalization using CIFAR-10 specific statistics.
  • ViT-Tiny (Transformer): Upscales images to 224 x 224 to align with pre-trained patch embeddings and utilizes ImageNet-1K normalization parameters.

Backbone & Classification Head

We adopt a Transfer Learning strategy by utilizing state-of-the-art architectures pre-trained on the ImageNet-1K dataset. Each model consists of a feature-extracting Backbone and a specialized Classification Head.

ResNet50 (CNN)
  • Backbone: Uses 50 layers of residual blocks to extract local spatial features. It is highly efficient for 32 x 32 images due to its inherent Spatial Inductive Bias.
  • Head: A Global Average Pooling (GAP) layer followed by a single Linear Layer (2048 -> 10), mapping deep features to the 10 CIFAR-10 classes.
ViT-Tiny (Vision Transformer)
  • Backbone: Processes the image as a sequence of 16 x 16 patches. It captures Global Context through Multi-Head Self-Attention. We chose the Tiny variant to optimize performance within our 16GB RAM constraint.
  • Head: Uses the final state of the [CLS] token (dimension 192) fed into a Linear Layer (192 -> 10) to produce the final class logits.

Output

The final output of the pipeline is a 10-dimensional probability vector. We evaluate the models based on their ability to generalize to the unseen Test Set. Based on our 10-epoch training run, we recorded the following metrics:

  • Best Val Accuracy: The peak classification success rate achieved during the 10-epoch cycle.
  • Macro-F1 & Micro-F1: Balanced metrics that account for precision and recall across all 10 classes.
  • Avg. Time per Epoch: A critical measure of computational efficiency and hardware utilization.

Based on the empirical evidence gathered, we can draw three major conclusions regarding the architectural trade-offs:

  • Performance Gap: ResNet50 (CNN) achieved a higher Validation Accuracy (80.56%) than ViT-Tiny, demonstrating the superior inductive bias of CNNs for local feature extraction on small-scale, low-resolution datasets.
  • Computational Trade-off: Despite having fewer parameters (5.7M vs. 23.5M), ViT-Tiny required over 2.2x more training time per epoch (~85s) due to the computational overhead of upscaling images to 224 x 224.
  • Architectural Efficiency: ResNet50 remains the optimal choice for CIFAR-10, providing a better balance between accuracy, training speed, and resource utilization on consumer-grade hardware.

4. Error Analysis

While the aggregate accuracy provides a high-level performance metric, a detailed error analysis reveals specific semantic confusion zones. By examining the Confusion Matrix, we identify which classes are mutually confused and correlate these errors with the visual characteristics of the CIFAR-10 dataset at its native 32 x 32 resolution.

Figure 6: Class-wise distribution of False Negatives (Missed) and False Positives (Wrongly Predicted) for the ResNet50 model.

Figure 7: Confusion Matrix of ResNet50 on the CIFAR-10 test set, illustrating inter-class misclassifications.

4.1 The Biological "Semantic Sink" (Cat vs. Dog)

The most significant error cluster in the entire system occurs between the Cat and Dog classes.

  • Observation: The matrix shows bidirectional confusion: 175 Dogs were misclassified as Cats, and 118 Cats were misclassified as Dogs.
  • Analysis: At 32 x 32 resolution, fine-grained biological details (fur texture, whisker shape, eye color) are blurred into indistinguishable pixel noise. The CNN backbone is forced to rely on overall organic contours and postures, which are remarkably similar for these two domestic quadruped species.

4.2 Structural Ambiguity in Transportation (Automobile vs. Truck)

A secondary, notable error zone exists within the transportation categories, specifically between Automobile and Truck.

  • Observation: The matrix reveals significant bidirectional confusion: total 129 instances (59 Automobiles confused as Trucks and 70 Trucks confused as Automobiles).
  • Analysis: The model struggles to differentiate rigid, man-made transportation shapes when visual overlaps in structural features occur. At low resolution, metallic textures, wheels, and silhouettes can look identical. The primary difference—cargo bed shape—can be lost if the perspective is ambiguous.

4.3 Organic Contour Confusion (Deer confused as Birds)

A secondary organic error cluster is observed where the model fails to differentiate non-rigid silhouettes.

  • Observation: The matrix shows 87 Deer misclassified as Birds.
  • Analysis: This error likely stems from silhouette overlap, particularly when fine-grained details are lost. Both Deer and Birds can present complex, non-rigid organic contours (legs vs. wings, heads vs. beaks) against complex backgrounds. A Deer viewed from a distance might have a shape profile similar to a large Bird at low resolution.

4.4 Strongest Categories (Frog, Automobile, Ship)

The model achieves maximum reliability in classes with highly distinct, non-overlapping visual features.

  • Observation: The highest number of correct predictions are for Frog (884), Automobile (875), and Ship (871).
  • Analysis: These classes have unique color profiles (green for frogs) and geometric structures that do not share semantic space with others in CIFAR-10, making their features exceptionally reliable for the CNN.

7. Conclusion

Key Findings & Insights

Convolutional Superiority (ResNet50)

Convolutional-based models like ResNet50 provide absolute peak accuracy for low-resolution datasets by leveraging Spatial Inductive Bias. By focusing on local pixel neighborhoods, ResNet successfully extracts critical features from 32 x 32 images, successfully mitigating the information loss that occurs when global attention is spread too thin.

Transformer Adaptation (ViT-Tiny)

Despite the dominance of Transformers in large-scale vision, ViT-Tiny proved to be a high-latency candidate for this specific task. The requirement for 224x224 upscaling to satisfy patch embedding logic resulted in a 2.2x increase in training time, proving that global self-attention outweighs local convolutions only when extreme data scale and resolution are present.
